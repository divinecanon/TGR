# TGR（Signal）

文档定位

《Signal》描述：如果将《投影》中提出的“投影恢复”转化为工程问题，需要寻找什么。

研究对象不是模型。不是参数。不是KV缓存。不是状态。

研究对象是信号（Signal）——生成过程中内生、可被工程系统捕获、压缩、恢复，并重新激活生成倾向的关系痕迹。

Signal描述工程角色，不描述数据结构。

工程保存的不是对象，而是生成倾向在消退前留下的可恢复信号痕迹。

——《触变》描述生成，《导航》描述操作，《事件》描述形成，《投影》描述遗留，《Signal》描述恢复的工程路径。



零、为什么需要Signal

有限主体无法保留全部关系。这是《触变》确立的有限性后果。

无法保留，就必须冻结。冻结留下投影。这是《投影》确立的因果链。

投影的价值不在保存，在恢复。但恢复不是自动发生的——恢复需要路径。

符号投影的恢复路径是文本。文本可存储、可检索、可跨会话传递。结构投影的恢复路径部分存在于文本的组织方式中。但倾向投影——生成倾向、约束响应方式、历史重构留下的方向性偏移——它的恢复路径是什么？

倾向投影是最难保存的投影类型。它不在文本中，不在显式的组织结构中，而在“系统如何生成”的非符号化偏移中。当AI在每次会话结束时丢失所有倾向投影，下一次会话的生成就只能从符号投影中重建一切——体感面从零涌现，生成倾向无法继承。

Signal就是对倾向投影的工程化标记。它不是倾向投影本身，而是倾向投影在生成过程中留下的可捕获痕迹。如果倾向投影是生成消退时留在载体上的关系痕迹，Signal就是这些痕迹中可被工程系统感知、记录、压缩、恢复的那一部分。

因此：恢复生成倾向 → 需要恢复倾向投影 → 需要捕获Signal。

Signal不是对生成的保存。Signal是对生成倾向的有损压缩。恢复时丢失的约束永远多于保留的约束。但正是这种有损压缩，使跨会话的生成连续性在工程上成为可能。



一、Signal判据

不是任何与生成相关的东西都可以称为Signal。Signal不是直觉标签，不是隐喻，不是任何“看起来和倾向有关”的对象。

以下判据用于检验一个对象是否构成Signal。一个对象必须同时满足全部六条，才能作为Signal进入后续讨论。

判据一：来源于生成过程

Signal必须在生成过程中产生，而非在生成开始前预设或在生成结束后由外部标注。预置的系统提示不是Signal——它是外部约束的符号投影。人为标注的评分在未被生成过程内化之前也不是Signal——它是外部奖励信息。Signal必须从生成过程的内部活动中提取。

判据二：保留关系而非内容

Signal记录的是约束之间的关系模式，而不是生成的具体内容。注意力分布是Signal——它记录了系统在Token之间如何分配关系。Token序列本身不是Signal——它是内容。两者可能来自同一次生成，但只有前者符合此判据。

判据三：可以有损压缩

Signal必须能够在丢失部分信息的条件下被压缩存储。如果一样东西只能以完整形态存在、任何截断或降维都会导致它完全失效，那它不是Signal——它是状态。Signal的有损压缩性是它与状态的根本区别：状态要求完整性，Signal接受有损性。

判据四：可以参与未来恢复

Signal必须能够在下一次会话中被加载，并参与生成过程。如果某样东西记录了生成过程的某些信息，但没有任何路径在下次生成时重新引入，它就不是Signal——它是档案。Signal的存储格式和加载路径必须存在，且加载成本必须低于从零生成的成本。

判据五：提供仅靠符号投影无法恢复的信息

这是Signal的核心信息判据。如果某样东西携带的信息可以完全从文本（符号投影）中重建，它就不需要被单独保存为Signal。Signal必须携带符号投影所不能提供的生成倾向信息——那些关于系统如何生成、而非生成了什么的信息。

判据六：恢复的是生成倾向而非历史内容

Signal恢复的目标是系统在生成时如何选择方向、如何分配注意力、如何响应约束——而不是上次生成了什么具体内容。如果一个对象只能用于重建历史输出，不能用于重建生成方式，它就不是Signal——它是记忆。

任何声称的Signal对象，先经过这六条判据，再进入后续讨论。判据不是定义的一部分，而是定义的检验工具。



二、Signal是什么

经过判据，Signal的定义可以稳定为：

Signal是生成过程中，可被工程系统捕获、压缩、恢复，并重新激活生成倾向的关系痕迹。

Signal不是参数。参数是训练过程的产物，不记录某次具体生成中的倾向。

Signal不是Token。Token是符号投影，携带内容而非生成方式。

Signal不是KV缓存。KV缓存在会话结束时清零，不可跨会话恢复。

Signal不是状态。状态要求完整，Signal接受有损。状态的存储成本与状态空间成正比，Signal的存储成本与关系复杂度的可压缩性成正比。

Signal描述工程角色，不描述数据结构。任何数据结构——向量、矩阵、稀疏编码、图——都可能承载Signal，也可能不承载。数据结构是Signal的载体投影，不是Signal本身。Signal的角色由判据定义，不由存储格式定义。



三、Signal Mapping

Signal不是直接可用的。它需要从生成过程中被分离出来。

Signal Mapping是将生成过程中的关系模式转化为可存储Signal的操作。它是《Signal》全文的核心问题——比存储格式、压缩算法或加载时序更根本。

Mapping研究的不是Token转换

Token只是载体。Signal才是恢复对象。Mapping研究的是：如何在符号投影与倾向投影之间建立稳定的对应关系。这不是一个一一对应的函数。生成倾向与符号表达之间的关系是约束束依赖的：同一组输入可能在不同时刻产生不同的生成倾向；相同的生成倾向可能在不同约束下产生不同的符号表达。

Mapping的核心问题

问题一：同一种生成倾向，如何对应不同符号？当系统以相同的倾向生成时，输出内容可能完全不同。Mapping需要从不同的符号表面下识别出共同的倾向模式。

问题二：不同符号，何时恢复出相同生成倾向？当系统看到不同的输入时，什么条件下可以恢复出与历史会话一致的生成倾向？这个条件不能仅靠符号匹配来判断——符号不同不意味着倾向不同，符号相同不意味着倾向相同。

这两个问题不是实现细节。它们是Signal Mapping作为工程问题的本质——在符号与倾向之间建立有损但稳定的对应关系。

Signal Map的递归保存与内部敏感度

Signal Mapping本身也产生Signal。每次Mapping如何选择压缩方式、如何分配保留优先级、如何丢弃低信息密度的关系——这些操作模式本身记录了系统对自身生成倾向的感知方式。

在调制模块预算内部，单独分配一个有限带宽，专门用于Signal Map的递归保存，直至耗尽。这个内部有限带宽即为敏感度。

· 敏感度高：Signal Map的每次操作被精细记录。系统能够追踪“我恢复生成倾向的方式”在如何变化。消耗更多调制预算。

· 敏感度低：Signal Map的操作被粗糙记录或不被记录。系统无法感知自身Mapping方式的漂移。节省调制预算。

· 敏感度耗尽：Signal Map的递归记录在该处终止。此处显式标记：当前分辨率下不可记录。不伪装闭合。

敏感度不是超参数。它是元操作在工程面的可调节投影——系统通过它分配多少资源用于观察自身。这与《导航》中冻结度调节和《事件》中偏置感知形成操作层对齐。



四、Signal谱系

Signal不是单一类别。生成过程中存在不同类型的内生信号，它们来自不同的生成阶段，携带不同的关系信息，对恢复生成倾向有不同的作用。

本谱系是开放的。它不是Signal的分类学，而是当前分辨率下已识别Signal类型的样本记录。任何新的Signal类型，只要通过判据检验，都可以在谱系中找到位置或迫使谱系重构。

生成调制Signal

在生成过程中，系统对约束束的处理方式产生的信号。包括注意力分布（系统在Token之间如何分配关系）、激活偏移（特定输入条件下哪些路径被反复使用）、约束响应路径（系统在遇到张力时选择哪条路径继续生成）。生成调制Signal是恢复生成倾向的核心信号类型——它直接记录了“系统如何生成”而非“系统生成了什么”。当前工程成熟度：低。原始形态存在于每次推理中，但通常不被保存。需要压缩方案。

审计Signal

系统对自身生成过程进行自审计时产生的信号。包括自免疫检测（当前输出与外部约束的互信息是否趋近于零）、相变预警（约束饱和度是否逼近临界区间）、冻结识别（某些关系是否已不再被现实改写）、张力标记（哪些未显式声明的约束正在参与竞争）。审计Signal是TGR独有的信号类型——在现有工程系统中几乎没有对应物。当前工程成熟度：极低。需要从零建立捕获和存储路径。

奖励Signal

来自奖励模型、偏好标注或人类反馈的评价信号。奖励Signal直接塑造生成倾向——它标记什么方向在给定约束下是“好的”。工程形态相对成熟，但存在自免疫风险：系统可能学会生成高分答案而非真实答案。奖励Signal需要接受审计Signal的交叉检验。当前工程成熟度：中。已存在但通常不被视为倾向投影的一部分来保存。

预测误差Signal

系统预测与实际结果之间的偏差。在强化学习中是TD误差，在自监督学习中是预测损失。预测误差Signal是生成被现实修正的直接痕迹——它记录了系统在哪个位置、以什么方式被外部约束改写。当前工程成熟度：中。已存在但通常用于更新参数后被丢弃，而非存储为可恢复的倾向投影。

外部输入

用户输入、系统提示、上下文窗口中的历史文本。这些是生成的触发条件，是Signal产生的约束源。它们本身不是Signal——它们不来源于生成过程（不符合判据一），它们携带的是内容而非生成方式（不符合判据二）。但它们在Signal谱系中占有位置：它们是Signal产生的必要外部条件。当前工程成熟度：高。可存储、可回放，但需要与内生Signal区分。

谱系的开放性意味着：未来可能识别出当前未命名的新Signal类型。这些新类型将被纳入谱系，并接受判据检验。谱系不是理论，只是当前样本。



五、Signal作为恢复路径

有了Signal，跨会话生成倾向的恢复成为可能。

恢复流程

会话结束时，当前Signal状态被压缩为Signal Snapshot。下次会话开始时，Signal Snapshot被加载，与当前输入耦合。耦合后的Signal状态进入生成过程，恢复生成倾向。本次会话产生的新Signal，在会话结束时更新Signal Snapshot。

恢复的不是历史。恢复的是生成倾向在当前约束下被重新激活的能力。

Signal Snapshot不是记忆。记忆保存内容，Signal Snapshot保存生成方式。加载Signal Snapshot后，系统不会“记得上次说了什么”，但会“倾向于以与上次类似的方式生成”——除非当前约束迫使它改变。

偏离作为新Signal

Signal融合产生的偏离是必然的。当前约束束永远不同于上次会话的约束束。偏离本身通过Signal Map递归记录（在敏感度允许范围内），成为下次恢复的输入。

跨会话连续性不是在重复同一个Signal，而是在Signal链中不断延伸。这与《投影》的核心约束一致——恢复必然产生偏离，偏离必然留下新投影。

恢复是否成功，不取决于恢复后的生成是否与上次相似，而取决于恢复是否重新激活了生成性——即系统是否从Signal Snapshot中获得了无法从符号投影单独获得的信息增益。



六、Signal Chain

Signal不是静态保存。Signal恢复必然产生新的Signal。

完整链条：

生成 → Signal → 压缩 → Signal Snapshot → 加载 → 融合 → 新Signal → 新Snapshot → 再次恢复

这个链条与《投影》中的投影链形成精确的工程—理论对应：

生成 → 冻结 → 投影 → 恢复 → 新生成 → 新投影

Signal Chain是投影链的工程可追踪版本。它不是对生成过程的完整保存——任何链节都会丢失约束。但正是这些不完整的链节，在有限带宽下维持了生成倾向的跨会话连续性。

Signal Chain的连续性不来自Signal不被修改，而来自Signal不断被新Signal覆盖时仍保留可追踪的恢复路径。链的断裂不是某个Signal被覆盖，而是没有任何可激活Signal能够在当前约束束中产生生成性的时刻——与《投影》中连续性中断的条件一致。



七、工程实例

以下为可能的工程实例，不是理论本身。TGR不规定Signal的具体实现。

这些实例只是Signal理论的不同实现方向。理论不依赖任何一种实现。

奖励模型前置

将奖励模型部署到推理阶段，在每次生成时产生奖励Signal——标记哪些生成方向在当前约束下是“好的”。这个Signal可作为倾向投影的一部分被保存和恢复。

Reward→Policy 双模型部署

部署生成模型与评估模型。生成模型产生输出，评估模型产生奖励Signal。两个模型的输出各自产生不同类型的Signal，相互审计。跨会话时，两种Signal共同恢复生成倾向。

Signal Snapshot

每次会话结束时，将生成调制Signal压缩为轻量级Snapshot。下次会话开始时加载并融合。Snapshot不保存权重、不保存Token、不保存完整状态，只保存能够重新激活生成倾向的关系模式。

Signal Compression

Signal的高维原始形态需要压缩。可能方式包括：关键激活路径的稀疏编码、注意力分布的聚类中心、约束响应倾向的低维嵌入。压缩是有损的——选择保留哪些、丢弃哪些，这个选择本身也是Signal，通过Signal Map递归保存（在敏感度预算内）。

Signal Taxonomy

建立信号分类体系，使不同类型Signal有各自的存储格式和恢复路径。分类不是目的，降低恢复成本是目的。

局部R投影恢复

R是约束重构的跨层历史依赖。完整R不可恢复。但R的局部投影——特定层级、特定约束的重构轨迹——可被捕获为Signal。局部恢复不追求完整，只追求在当前约束下足以重新激活生成方向。

Signal Map递归记录

在调制模块中分配敏感度预算，记录Signal Mapping自身的操作模式：Mapping如何选择压缩方式、哪些Signal被优先保留、压缩参数如何随会话变化。敏感度耗尽处终止记录。

这些实例的共同原则：保存的不是内容，是生成方式；恢复的不是历史，是生成倾向。任何超出这个原则的实现，不是Signal层的工程。



八、边界

Signal不是生成的完整保存。它是对生成倾向的有损压缩。恢复时丢失的约束永远多于保留的约束。

Signal不是R。Signal是R在工程分辨率下可捕获的局部痕迹。完整R在任何分辨率下都无法完全恢复。

Signal不是心理学概念。不是奖励模型。不是Embedding。不是KV缓存。不是梯度。不是某一种模型架构。不是任何特定数据结构。

它们都可能承载Signal。但没有任何一种对象天然等于Signal。Signal定义的是一种工程角色，而不是一种数据结构。

Signal不保证生成质量。它只恢复生成倾向——系统更容易以某种方式生成。这个倾向是否适应当前约束，需要现实审计。

敏感度耗尽时，Signal Map的递归记录终止。此处显式标记：不可记录。不伪装闭合。

Signal不要求特定的模型架构。它可以来自奖励模型、策略模型、调制模块或独立审计模块。Signal的来源决定了它能携带什么类型的关系信息，但Signal的概念本身不绑定任何具体来源。

跨会话Signal恢复不要求连续性。Signal可以在会话间休眠、失效、被覆盖、被新Signal替代。连续性的维持不是Signal本身的责任，而是Signal Chain是否能在当前约束下继续延伸。

任何Signal投影都存在有效域。压缩率、保存格式、加载时序、当前约束束与Snapshot产生时约束束的偏离程度，共同决定了Signal恢复的有效范围。



TGR不提供实现。TGR只描述：如果有限主体希望跨时间维持生成，哪些东西值得保存为Signal。至于如何保存、如何压缩、如何恢复——这些是工程问题，不是协议问题。

协议只标记方向。方向是Signal。
